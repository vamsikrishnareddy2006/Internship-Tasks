about this project

Why DevPulse exists

A full-stack demonstration of connecting a trained machine learning model to a real, working web application — end to end, with nothing mocked.

aim

The aim of DevPulse is to help development teams monitor the health of their codebase and commit activity in real time. By combining a machine learning model trained on developer activity metrics — commit frequency, code review engagement, test coverage, and late-night commits — with a Flask backend and an interactive interface, DevPulse automatically flags risky commit patterns (Low, Medium, or High risk) before they lead to bugs, technical debt, or production issues. Beyond prediction, the project demonstrates complete full-stack integration: a trained model connected to a live web app, historical predictions persisted in a database, and results presented through a clear, purpose-built interface.
